## P0（高优先级，建议立即处理）

- [ ] 修复配置项未持久化问题（应用白名单/忽略列表、网站忽略列表）
  - 问题：右键菜单中修改配置后，部分路径没有调用 `appConfig.Save()`，重启后配置丢失。
  - 影响：用户误以为配置生效，实际数据不一致。
  - 建议：在所有修改 `config.Behavior.*` 的操作后统一调用保存；补充最小回归测试（修改后重启仍生效）。

- [ ] 修复 ViewModel 跨线程更新 UI 绑定数据问题
  - 问题：多处 `Task.Run` 内直接写入绑定属性（如 `Data`/`MonthData`/`YearData` 等）。
  - 影响：可能触发 WPF 跨线程异常、界面偶发错乱或数据不刷新。
  - 建议：后台线程只做数据计算；属性赋值切回 UI 线程（`Dispatcher` / `SynchronizationContext`）；逐步将 `async void` 改为 `async Task`。

## P1（中高优先级，建议尽快处理）

- [ ] 完善全局输入钩子与系统事件生命周期管理
  - 问题：键盘/鼠标钩子与系统事件注册后，停止或退出路径中释放不完整。
  - 影响：潜在资源泄漏、重复注册、长期运行稳定性下降。
  - 建议：为 `Sleepdiscover`、`InputServicer` 增加明确的 `Stop/Dispose` 释放逻辑，并在程序退出时统一回收。

- [ ] 为更新器增加下载完整性校验与更安全的升级流程
  - 问题：更新包下载后直接解压覆盖，无 hash/签名校验；存在直接 `Kill` 主进程行为。
  - 影响：供应链安全风险、升级中断导致文件状态不一致。
  - 建议：
    - 使用 SHA-256 校验（或签名校验）；
    - 优先请求主程序优雅退出，必要时再兜底强制结束；
    - 解压失败时保留可恢复路径和明确错误日志。

- [ ] 统一构建与依赖恢复流程（保证新环境可复现）
  - 问题：当前 `dotnet build Tai.sln` 失败，主要受 `packages.config` + 本地 `packages` 目录依赖影响。
  - 影响：开发者新环境无法直接构建，CI 落地困难。
  - 建议：
    - 在 README 增加标准构建步骤（NuGet restore + MSBuild）；
    - 提供一键脚本（如 `build.ps1`）统一还原与构建；
    - 中长期考虑迁移到 `PackageReference`。

## P2（中优先级，建议纳入近期迭代）

- [ ] 清理“吞异常”代码路径，补齐日志上下文
  - 问题：存在 `catch {}` / `catch` 后无日志的分支。
  - 影响：线上问题难以定位，增加维护成本。
  - 建议：至少记录异常消息 + 场景上下文（模块、输入参数、关键状态）。

- [ ] 修复 Logger 的并发访问竞态
  - 问题：共享 `loggers` 列表在多线程 Add/Clear/拼接下缺乏统一同步。
  - 影响：日志丢失、偶发异常、崩溃后关键信息缺失。
  - 建议：改为线程安全队列（如 `ConcurrentQueue<string>`）或统一加锁策略。

- [ ] 重构 Database 读写锁模型（轮询阻塞 -> 更可靠同步）
  - 问题：通过 `Thread.Sleep(1000)` 轮询等待读写状态。
  - 影响：吞吐低、等待不可控、时序问题隐蔽。
  - 建议：使用 `ReaderWriterLockSlim` 或明确事务/上下文生命周期管理，减少阻塞等待。

## 可选优化（P3）

- [ ] 清理调试输出与无用 using
  - 问题：项目中存在较多 `Debug.WriteLine` 与冗余 `using`。
  - 影响：可读性下降，发布版本日志噪音偏多。
  - 建议：统一日志等级和开关；使用分析器自动清理无用引用。

- [ ] 统一异步风格与取消机制
  - 问题：`async void` 与无 `CancellationToken` 的后台任务较多。
  - 影响：任务不可控，退出过程可能拖尾。
  - 建议：核心流程改为 `async Task` + `CancellationToken`，并在关闭时发出取消信号。
